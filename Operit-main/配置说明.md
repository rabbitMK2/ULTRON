# 项目配置说明

## 已完成的配置

✅ **本地 Maven 仓库配置** - 已配置 `local-m2` 目录用于手动下载的依赖  
✅ **Aliyun 镜像配置** - 已添加 Aliyun Maven 镜像作为备用源

## 需要手动完成的步骤

### 1. 初始化 Git 子模块（必需）

项目使用了 Git 子模块来管理外部依赖。**必须在 Android Studio 同步之前执行此步骤**：

#### 方法 A：使用 Git 命令（推荐）

在项目根目录打开终端（PowerShell 或 CMD），执行：

```bash
git submodule update --init --recursive
```

如果您的项目不是通过 Git 克隆的，或者子模块未正确初始化，请手动克隆：

#### 方法 B：手动克隆子模块

1. **克隆 MNN 仓库**：
```bash
cd "D:\BaiduNetdiskDownload\奥创最终版\Operit-main\mnn\src\main\cpp"
git clone https://github.com/alibaba/MNN.git MNN
```

2. **克隆 Terminal 仓库**（如果 terminal 目录为空）：
```bash
cd "D:\BaiduNetdiskDownload\奥创最终版\Operit-main"
git clone https://github.com/AAswordman/OperitTerminalCore.git terminal
```

### 2. 下载 Google Drive 依赖包（如需要）

根据 `README.md` 和 `docs/BUILDING.md`，项目需要从 Google Drive 下载以下依赖包：

- `app/src/main/assets/models/` - 模型文件
- `app/src/main/assets/subpack/` - 子包文件  
- `app/src/main/jniLibs/` - 原生库文件

如果这些目录不存在或为空，请参考项目文档中的 Google Drive 链接下载。

### 3. 在 Android Studio 中同步

完成上述步骤后，在 Android Studio 中：
1. 点击 **File → Sync Project with Gradle Files**
2. 等待同步完成
3. 如仍有错误，点击 **File → Invalidate Caches / Restart → Invalidate and Restart**

## 当前已知问题

⚠️ **Gradle 源分发警告**：虽然添加了禁用选项，Android Studio 可能仍会尝试下载 Gradle 源代码。这些警告**不影响实际构建**，只会影响 IDE 的 Kotlin DSL 代码补全功能。构建 APK 不受影响。

⚠️ **SSL 证书问题**：如果遇到 SSL 证书相关错误，可能是网络环境问题。已配置 Aliyun 镜像作为备用，应该能够解决大部分依赖下载问题。

## 验证配置

配置完成后，您应该能够：
1. ✅ 在 Android Studio 中成功同步项目
2. ✅ 编译项目（虽然可能还需要下载依赖包）
3. ✅ 构建 Debug APK

如果仍有问题，请检查：
- Git 子模块是否已正确初始化
- 网络连接是否正常
- Android SDK 和 NDK 是否已正确安装
